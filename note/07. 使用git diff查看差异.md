# 使用 git diff 查看差异

`git diff` 是 Git 中用于**查看文件差异**的核心命令，能清晰对比不同区域、不同版本、不同分支的代码变化。

## 核心作用

* 查看文件在**工作区、暂存区、版本库**之间的差异
* 查看**两个提交版本**之间的代码差异
* 查看**两个分支**之间的代码差异

> 补充说明：日常开发常用图形化工具（GUI）查看差异，但学习 `git diff` 命令非常必要。在无GUI的服务器环境中，只能通过命令行操作Git。

---

## 常用命令详解

### 1. 工作区 ↔ 暂存区（默认）

不带任何参数，默认对比**工作区**和**暂存区**的差异，显示未暂存的修改内容

```bash
git diff
```

### 2. 工作区 ↔ 版本库

对比**工作区**和**当前最新版本库（HEAD）** 的所有差异

```bash
git diff HEAD
```

### 3. 暂存区 ↔ 版本库

对比**暂存区**和**版本库**的差异，查看已暂存、待提交的修改

```bash
# 标准写法
git diff --cached
# 等价写法（Git 新版本可以使用）
git diff --staged
```

### 4. 对比两个指定版本

对比两个历史提交版本之间的差异，支持提交ID / HEAD相对引用

```bash
# 方式1：使用提交ID（使用 git log --oneline 命令查看）
git diff <提交ID1> <提交ID2>

# 方式2：使用HEAD简化（对比最新版本和上一个版本）
git diff HEAD HEAD^

# 方式3：混合使用HEAD和提交ID
git diff <提交ID> HEAD^
```

### 5. 对比两个分支

对比两个不同分支之间的代码差异

```bash
git diff <分支名1> <分支名2>
```
