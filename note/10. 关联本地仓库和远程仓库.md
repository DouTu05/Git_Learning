# Git 远程仓库（GitHub）操作：拉取、提交、合并
本笔记包含 **本地关联远程仓库、克隆拉取代码、推送提交代码、远程与本地合并** 全套操作，适配 GitHub 平台。
> 相较于视频内容进行了一定的拓展，便于开发使用。

---

## 一、核心概念
- **远程仓库**：GitHub 上托管的代码仓库（云端）
- **本地仓库**：你电脑上的 Git 仓库
- **`origin`**：Git 默认的远程仓库别名（固定写法，无需修改）
- **`main/master`**：仓库默认主分支（GitHub 新仓库默认为 `main`）

---

## 二、场景1：本地已有仓库 → 关联 GitHub 远程仓库
先在本地写了代码，现在要同步到 GitHub。

### 1. GitHub 端操作
1. 打开 GitHub，点击 **New repository** 新建仓库
2. 填写仓库名，无需初始化文件，直接创建
3. 复制仓库的 **HTTPS 地址**

### 2. 本地终端操作
```bash
# 1. 查看本地是否关联远程仓库
git remote -v

# 2. 关联远程仓库（替换为你的GitHub仓库地址）
git remote add origin https://github.com/xxx/xxx.git

# 3. 再次查看，验证关联成功
git remote -v
```

---

## 三、场景2：从 GitHub 克隆（拉取）已有仓库
GitHub 已有仓库，你要下载到本地。

### 实操命令
```bash
# 克隆远程仓库到本地（自动完成关联）
git clone 远程仓库地址

# 示例
git clone https://github.com/xxx/xxx.git
```
> 执行后，会在当前文件夹生成项目目录，**无需手动关联远程**。

---

## 四、提交本地代码到远程仓库（推送）
本地完成 `add` + `commit` 后，将代码推送到 GitHub。

### 基础命令
```bash
# 首次推送（关联本地主分支与远程主分支）
git push -u origin main

# 后续推送（直接执行）
git push
```

### 完整流程
```bash
# 1. 本地修改、添加、提交
git add .
git commit -m "新增功能/修复bug"

# 2. 推送到远程
git push
```

---

## 五、拉取远程最新代码（同步）
当远程仓库代码更新（他人提交/网页修改），本地需要同步。

```bash
# 拉取远程最新代码，并自动与本地合并

# 完整写法（指定远程仓库和分支）
git pull <远程仓库名> <远程分支名>:<本地分支名>

# 简化写法（当远程分支名和本地分支名相同时，可省略冒号后部分，最常用）
git pull <远程仓库名> <远程分支名>

# 最简写法（已关联分支时，直接执行）
git pull
```
> 等价于：`git fetch`（拉取代码）+ `git merge`（合并代码）

---

## 六、远程与本地合并（冲突解决）
### 1. 什么是合并冲突？
本地和远程**修改了同一个文件的同一行代码**，Git 无法自动合并，会提示冲突。

### 2. 冲突处理步骤
1. 执行 `git pull` 拉取代码，提示冲突
2. 打开冲突文件，手动编辑代码（保留需要的内容）
3. 标记冲突已解决
```bash
git add 冲突文件名
```
4. 提交合并结果
```bash
git commit -m "解决远程与本地的合并冲突"
```
5. 推送到远程
```bash
git push
```

---

## 七、常用命令总结
| 命令 | 作用 |
| :--- | :--- |
| `git remote add origin 地址` | 本地关联远程仓库 |
| `git remote -v` | 查看远程仓库信息 |
| `git clone 地址` | 克隆/拉取远程仓库到本地 |
| `git push` | 本地代码推送到远程 |
| `git pull` | 拉取远程代码并合并到本地 |

---

## 八、注意事项
1. **首次推送必须加 `-u`**：绑定本地与远程分支，后续直接用 `git push`
2. **推送前必须先拉取**：多人协作时，先 `git pull` 同步代码，避免冲突
3. **冲突不可怕**：手动修改代码后，重新提交推送即可
4. GitHub 仓库默认分支：`main`（老项目为 `master`）
