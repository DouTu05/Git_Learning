# Git 基础：将文件添加到本地仓库
Git 管理文件的核心三步流程：
**查看仓库状态 → 添加文件到暂存区 → 提交到本地仓库**

### 核心命令速查
| 命令 | 作用 |
| :--- | :--- |
| `git status` | 查看仓库/文件的当前状态 |
| `git add` | 将文件添加到**暂存区** |
| `git commit` | 将暂存区文件提交到**本地仓库** |
| `git log` | 查看提交历史记录 |

---

## 一、git status：查看仓库状态
`git status` 是 Git 最常用的命令之一，用于实时查看：
- 当前所在分支
- 文件的状态（未跟踪/已暂存/已修改）
- Git 给出的操作提示

### 实操演示
1. 创建一个文本文件并写入内容
```bash
echo "这是第一个文件" > file1.txt
```
2. 查看文件内容
```bash
cat file1.txt
```
3. 执行状态查看命令
```bash
git status
```

### 输出结果解析
```bash
On branch master        # 当前在 master 分支
No commits yet          # 仓库还没有任何提交记录
Untracked files:        # 【关键】未跟踪的文件（Git 不认识这个文件）
  (use "git add <file>..." to include in what will be committed)
        file1.txt       # 未跟踪的文件：file1.txt
```
状态说明：`file1.txt` 是**新文件**，还未被 Git 管理。

---

## 二、git add：将文件添加到暂存区
`git add` 的作用：把工作区的文件放入**暂存区**，为最终提交做准备。

执行后文件状态会从 `未跟踪` → `已暂存`。

### 1. 常用用法
```bash
# 1. 添加单个文件
git add 文件名

# 2. 添加多个文件（空格分隔）
git add 文件1 文件2 文件3

# 3. 添加当前目录下所有文件
git add .

# 4. 通配符添加（批量添加指定类型文件）
git add *.txt
```

### 2. 实操演示
#### 方式1：添加单个文件
```bash
git add file1.txt
```
执行后再次查看状态：
```bash
Changes to be committed:  # 【关键】已暂存，等待提交
        new file:   file1.txt
```

#### 方式2：通配符批量添加
1. 批量创建测试文件
```bash
echo "file2" > file2.txt
echo "file3" > file3.txt
echo "file4" > file4.txt
echo "file5" > file5.sh
```
2. 批量添加所有 `.txt` 后缀文件
```bash
git add *.txt
```
3. 查看状态：仅 `.txt` 文件被暂存，`.sh` 文件仍未跟踪。

---

## 三、git commit：提交到本地仓库
`git commit` 会将**暂存区的所有文件**永久保存到 Git 本地仓库，完成文件的正式托管。

### 核心规则
1. 必须搭配 `-m` 参数填写**提交信息**
2. 如果不指定 `-m` 这个参数，那么会进入一个交互式界面，默认会使用 vim 来编辑提交信息。
3. 仅提交暂存区文件，未 `add` 的文件不会被提交

### 实操演示
```bash
git commit -m "第一次提交：新增 file1.txt"
```

### 通用格式
```bash
git commit -m "自定义提交说明"
```

---

## 四、git log：查看提交记录
提交完成后，用该命令查看仓库的所有历史提交。

### 常用用法
```bash
# 查看完整提交记录（含作者、时间、哈希值）
git log

# 简洁版查看（仅显示提交ID + 提交信息）
git log --oneline
```
