# 将文件添加到仓库

* **git status**：查看仓库的状态
* **git add**：将文件添加到暂存区
* **git commit**：将暂存区中的文件提交到本地仓库

## git status

使用下面命令创建一个文件，并修改里面的内容
``` bash
echo "这是第一个文件" > file1.txt
```

创建完成后，可以通过下面命令来查看文件里的内容：
``` bash
cat file1.txt
```

这时，我们再执行 `git status` 命令来查看仓库的状态，会看到输出以下内容：
``` bash
$ git status
On branch master

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        file1.txt

nothing added to commit but untracked files present (use "git add" to track)
```

## git add

### git add通过文件名添加文件

 **git add**命令的作用就是将文件添加到暂存区，等待后续的提交操作。我们在终端中继续执行下面命令:
 ``` bash
 git add file1.txt
 ```
 执行成功后，再执行 `git status` 命令，会看到输出以下内容：
 ``` bash
$ git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
        new file:   file1.txt
```

* 我们可以使用以下命令将单个文件件添加到暂存区：
``` bash
git add <file>
```
* 我们也可以使用以下命令将多个文件添加到暂存区：
``` bash
git add <file1> <file2> <file3> ...
```
* 我们也可以使用以下命令将所有文件添加到暂存区：
``` bash
git add .
```
**git add**命令执行成功后，文件状态会变成已暂存(Staged)，表示文件已经添加到暂存区，等待后续的提交操作。

### git add使用通配符添加文件

**git add**命令还可以使用通配符来添加多个文件。通过执行下面命令，来创建多个文件用来测试通配符添加文件到暂存区。

``` bash
echo "file2" > file2.txt
echo "file3" > file3.txt
echo "file4" > file4.txt
echo "file5" > file5.sh
```

可以执行下面命令将所有 .txt 文件添加到暂存区里面
``` bash
git add *.txt
```
再通过`git status`命令可以看到所有的 .txt 文件都被添加到了暂存区。
``` bash
$ git status
On branch master
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   file2.txt
        new file:   file3.txt
        new file:   file4.txt

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        file5.sh
```

## git commit

我们可以使用**git commit**命令将暂存区中的文件提交到本地仓库。只有将文件提交到仓库中，我们的文件才算真正被Git管理起来。
> `git commit`命令在提交的时候需要使用 -m 参数来指定提交的信息，这个信息会被记录到仓库中。如果不指定 -m 这个参数，那么会进入一个交互式界面，默认会使用 vim 来编辑提交信息。
> 
> `git commit`提交时只会提交暂存区内的文件，而不会提交其他文件。
接下来通过执行下面命令来对 file1.txt 进行提交，同时添加信息为“第一次提交”

``` bash
git commit -m "第一次提交"
```

通用命令如下所示：

``` bash
git commit -m "message"
```

## git log

可以使用 **git log** 命令来查看提交记录。

``` bash
git log
```

可以使用 **--oneline** 参数来查看简洁的提交记录，这样就只显示每次提交的 ID 和提交的信息了。
``` bash
git log --oneline
```
