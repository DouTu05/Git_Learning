# Git 删除文件的两种方式
在Git中删除文件有两种常用方法，核心区别：是否需要手动同步暂存区。

---

## 一、方式1：系统命令 rm 删除 + git add 更新暂存区
`rm` 是**系统原生删除命令**，仅删除**工作区**的文件，不会自动同步到Git暂存区，需要手动执行 `git add` 更新。

### 操作步骤
1. **删除工作区文件**
```bash
# 系统删除命令：仅删除本地文件
rm 文件名
```
2. **同步删除操作到暂存区**
```bash
git add 文件名
# 或批量同步所有修改
git add .
```
3. **提交删除操作到版本库**
```bash
git commit -m "删除文件xxx"
```

### 实操示例
```bash
# 删除工作区的 file3.txt
rm file3.txt
# 将删除操作同步到暂存区
git add file3.txt
# 提交
git commit -m "删除 file3.txt"
```

---

## 二、方式2：Git 专属命令 git rm 直接删除
`git rm` 是**Git专用删除命令**，**一步完成两个操作**：
1. 删除工作区的文件
2. 自动将删除操作同步到暂存区
无需手动执行 `git add`，直接提交即可。

### 操作步骤
```bash
# 一步删除：工作区 + 暂存区
git rm 文件名
# 直接提交
git commit -m "删除文件xxx"
```

### 实操示例
```bash
# 直接删除 file2.txt 并同步暂存区
git rm file2.txt
# 提交删除操作
git commit -m "删除 file2.txt"
```

---

## 三、补充：仅删除暂存区，保留工作区文件
如果只想让Git**停止跟踪文件**，但保留本地文件（不删除），使用 `--cached` 参数：
```bash
git rm --cached 文件名
```

---

## 四、两种方式对比总结
| 命令                | 工作区文件 | 暂存区状态 | 额外操作 |
| :------------------ | :--------: | :--------: | :------: |
| `rm` + `git add`    |    删除    |  手动同步  | 需add |
| `git rm`            |    删除    |  自动同步  | 无需add |

---

## 五、git ls-files：查看暂存区文件列表
`git ls-files` 是Git专用命令，**用于查看暂存区（索引）中所有被跟踪的文件**，可以直观确认文件是否在暂存区、删除操作是否生效。

### 基础命令
```bash
git ls-files
```

### 实操示例（结合删除操作）
1. **删除文件前**，查看暂存区所有文件
```bash
git ls-files
# 输出：file1.txt  file2.txt  file3.txt
```

2. **执行删除操作后**，再次查看（验证暂存区已更新）
```bash
rm file3.txt
git add file3.txt
git ls-files
# 输出：file1.txt  file2.txt （file3.txt 已从暂存区移除）
```
