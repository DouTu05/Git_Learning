# .gitignore 文件
在 Git 中，**`.gitignore`** 文件用于定义需要忽略的文件/目录，Git 会自动跳过这些文件，不会将其加入暂存区和版本库。让仓库体积更小、结构更干净，避免提交无关/敏感文件。

## 一、需要忽略的文件原则
1. 系统/软件自动生成的文件
2. 编译产生的中间文件、可执行文件
3. 运行时生成的日志文件、缓存文件、临时文件
4. 包含密码、密钥、token 等敏感信息的文件

## 二、文件作用
在 `.gitignore` 中写入文件/目录的匹配规则，符合规则的文件将**不会被 Git 追踪和提交**。

---

## 三、实操演示
1. 创建测试日志文件
```bash
echo "some log" > access.log
```
2. 查看状态，文件处于未跟踪状态
```bash
$ git status
Untracked files:
        access.log
```
3. 再创建一个日志文件用于对比
```bash
echo "other log" > other.log
```
4. 创建 `.gitignore` 文件，写入需要忽略的文件名
```bash
echo "access.log" > .gitignore
```
5. 再次查看状态：`access.log` 已被忽略，不再显示
```bash
$ git status
Untracked files:
        .gitignore
        other.log
```
6. 提交所有文件
```bash
git add .
git commit -m "ignore file sample"
```
7. 查看暂存区/仓库文件，验证忽略成功
```bash
$ git ls-files
.gitignore
other.log
```
结果：`access.log` 未被提交，忽略生效。

## 四、常用匹配规则（通配符）
实际开发中常用通配符批量忽略文件，无需逐个写文件名：
```bash
# 忽略所有 .log 后缀的文件
*.log

# 忽略指定目录（如 temp 整个文件夹）
temp/

# 忽略所有 .tmp 结尾的临时文件
*.tmp

# 例外规则：不忽略 error.log（放在 *.log 之后生效）
!error.log

# 忽略当前目录下的 test.txt
/test.txt
```

## 五、忽略文件夹（目录）
在 `.gitignore` 中，**目录名 + /** 可以忽略整个文件夹，是开发中最常用的规则。

### 实操演示
1. 创建一个缓存文件夹 `cache/`
```bash
mkdir cache
```
2. 在 `.gitignore` 中添加忽略规则
```bash
echo "cache/" >> .gitignore
```
3. 查看状态：`cache/` 文件夹已被忽略，不会被 Git 追踪

### 常用忽略目录示例
```bash
# 忽略依赖包目录
node_modules/
vendor/
# 忽略编译输出目录
dist/
build/
# 忽略日志目录
logs/
# 忽略临时缓存目录
temp/
```

## 六、Git 重要特性：空文件夹不会被纳入版本控制
### 核心规则
**Git 本身不跟踪、不管理空文件夹**，即使手动创建空目录，`git add` 和 `git commit` 也不会将其提交到仓库。

### 原因
Git 追踪的最小单位是**文件**，没有文件的空目录无法被记录。

### 解决方法（必须保留空文件夹时）
在空文件夹中创建一个名为 **`.gitkeep`** 的空文件（约定俗成的命名），Git 会因为这个文件而跟踪整个目录。

#### 实操演示
```bash
# 创建空文件夹
mkdir empty_dir
# 进入文件夹，创建 .gitkeep 文件
touch empty_dir/.gitkeep
# 提交后，空文件夹就会被保留在仓库中
```

## 七、.gitignore文件匹配规则
* 空行或者以#开头的行会被Git忽略。一般空行用于可读性的分隔，#一般用作注释
* 使用标准的Blob模式匹配，例如:
  * 星号*通配任意个字符
  * 问号?匹配单个字符
  * 中括号[]表示匹配列表中的单个字符，比如:[abc]表示a/b/c
* 中括号[]表示匹配列表中的单个字符，比如:[abc]表示a/b/c
* 中括号可以使用短中线连接，比如:
  * [0-9]表示任意一位数字，[a-z]表示任意一位小写字母
* 感叹号!表示取反

---

## 八、注意事项
1. `.gitignore` **仅对未跟踪的文件生效**
   如果文件已经被 `git add` 提交到仓库，再添加忽略规则**不会生效**，必须先删除该文件并提交后，规则才会生效。
2. `.gitignore` 本身**需要提交到仓库**
   方便团队成员共享忽略规则。
3. 空文件夹无法被 Git 管理，需借助 `.gitkeep` 文件实现目录保留。
