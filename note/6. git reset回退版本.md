# git reset 命令
在日常开发中，若需要**撤销修改**或**回退到历史版本**，`git reset` 是最核心的命令。它的作用是：将当前分支重置到指定的提交版本，同时控制工作区、暂存区的文件状态。

---

## 一、版本回退的指定方式
回退版本，Git提供了两种便捷写法：
1. **提交ID**：通过 `git log --oneline` 查看的 commit 哈希值（精准回退）
2. **相对引用（推荐）**：基于当前最新版本 `HEAD` 快速回退
   - `HEAD^`：回退到**上一个提交版本**
   - `HEAD~数字`：回退到**前 N 个提交版本**（例：`HEAD~3` = 回退3个版本）

---

## 二、git reset 的三种核心模式
`git reset` 有三个参数，区别在于**对工作区、暂存区的处理方式不同**，其中 `--mixed` 是默认模式。

### 1. `--soft` 软回退
**特点**：仅回退提交记录，**保留工作区 + 暂存区的所有修改**
- 回退后，文件状态为「已暂存」，可直接重新提交
- 适用场景：想合并多次提交、重新修改提交信息

```bash
# 回退到上一个版本（软回退）
git reset --soft HEAD^

# 回退到指定ID版本（软回退）
git reset --soft <提交ID>
```

### 2. `--mixed` 默认回退
**特点**：回退提交记录 + 清空暂存区，**仅保留工作区的文件内容**
- 这是 Git 默认模式，可省略 `--mixed` 参数
- 适用场景：想重新编辑文件后，再add、commit

```bash
# 完整写法（回退上一个版本）
git reset --mixed HEAD^

# 简写写法
git reset HEAD^

# 回退3个版本
git reset HEAD~3
```

### 3. `--hard` 硬回退
**特点**：回退提交记录 + 清空暂存区 + **丢弃工作区所有未提交修改**
- 高危操作：未提交的代码会永久丢失！
- 适用场景：彻底放弃当前所有修改，强制回退到历史版本

```bash
# 回退到上一个版本（硬回退）
git reset --hard HEAD^

# 回退3个版本（硬回退）
git reset --hard HEAD~3

# 回退到指定ID版本（硬回退）
git reset --hard <提交ID>
```

---

## 三、核心对比总结
| 参数 | 回退提交 | 保留暂存区 | 保留工作区 | 风险 |
| :--- | :---: | :---: | :---: | :--- |
| `--soft` | √ | √ | √ | 无 |
| `--mixed`（默认） | √ | × | √ | 无 |
| `--hard` | √ | × | × | 高 |

---

## 四、补充说明
1. 查看提交ID/版本：`git log --oneline`（简洁版）
2. 相对引用通用规则：
   - `HEAD^` = `HEAD~1`（上一版本）
   - `HEAD^^` = `HEAD~2`（上两个版本）
3. **重要提醒**：
   - 若代码已推送到远程仓库，**禁止**使用 `git reset --hard`，会导致远程代码冲突！

---

## 五、紧急补救：误操作 `--hard` 的恢复方法
即使不小心执行了 `git reset --hard` 丢失了代码，**也可以抢救回来**！
Git 会记录所有的操作历史，我们可以用 `git reflog` 命令找回丢失的版本。

### 1. 核心命令
`git reflog`：查看**所有操作记录**（包括 `reset`、`commit`、`checkout` 等，即使提交被删除也能查到）

### 2. 完整恢复步骤
1. **查看操作日志**
```bash
git reflog
```
输出示例（会展示所有操作，找到误操作前的版本）：
```
a1b2c3d (HEAD) HEAD@{0}: reset: moving to HEAD^
d4e5f6g HEAD@{1}: commit: 第二次提交
7h8j9k0 HEAD@{2}: commit: 第一次提交
```

2. **复制目标版本的提交ID**
   找到你想恢复到的版本（比如误操作前的 `d4e5f6g`）

3. **执行恢复命令**
```bash
git reset --hard d4e5f6g
```

### 3. 关键注意事项
1. `git log` 看不到被删除的提交，但 **`git reflog` 可以**；
2. Git 会保留 `reflog` 记录约 **90天**，及时操作都能恢复；
